# Stage 48: World-First Discovery Protocol (2026-05-23)

## Non-negotiable truth condition

A “world-first method discovery” claim is allowed only if all three are true:
1. Method is mathematically distinct from known families (not just a parameterization of stacking/MoE/ensemble/meta-learning).
2. Prior-art audit finds no direct or equivalent formulation in audited primary sources.
3. Empirical evidence shows robust advantage under pre-registered tests.

If any condition fails, claim must be downgraded.

## Discovery strategy (next executable work)

1. Define a new method class with a structural novelty axis:
   - Non-convex decision geometry with provable identifiability constraints.
   - Explicit environment-shift topology term that cannot be reduced to scalar gating.
   - Fold-invariant regularizer tied to cross-environment error transport.

2. Build strict equivalence checks:
   - Show whether the new equations can be algebraically reduced to existing blend/stack/MoE forms.
   - If reducible, reject as not novel.

3. Prior-art search protocol:
   - Fixed query sets, domains, date stamp, inclusion/exclusion rules.
   - Classify matches as exact / equivalent / family-level.

4. Robust evaluation protocol:
   - Same LOEO folds as baseline.
   - Paired tests, bootstrap CI, seed stability, sensitivity checks.
   - Novelty claim forbidden unless performance and robustness pass thresholds.

## Pre-registered thresholds for “strong claim”

- Point gain: >= 0.30 RMSE over current champion in both scopes.
- Paired tests: p <= 0.05 in both scopes.
- Bootstrap gain CI: lower bound > 0.
- Seed stability: probability of outperforming champion >= 0.80.

If any threshold fails: report as promising variant, not world-first discovery.
